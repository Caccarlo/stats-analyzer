## 1. PANORAMICA

### Cosa fa
Web app che analizza i falli commessi e subiti di un giocatore di calcio usando le API interne di SofaScore. L'utente naviga per paesi → campionati → squadre → formazione → giocatore, oppure cerca direttamente un giocatore dalla barra di ricerca. Per ogni giocatore mostra: medie falli aggregabili per competizione, lista partite con dettaglio falli (chi ha fallito chi, il minuto, la zona del campo), mappa posizioni medie dei giocatori coinvolti.

### Stack
- **Frontend**: React 18 + TypeScript + Vite + Tailwind CSS
- **Backend**: Node.js + Express (proxy minimale verso SofaScore per bypassare CORS)
- **Monorepo**: cartella unica con `server/` e `client/`

### Perché serve il proxy
Le API SofaScore bloccano le richieste cross-origin (CORS). Il browser impedisce a `localhost:3000` di chiamare `sofascore.com`. Il server Express fa da passacarte: il browser chiama `localhost:3001`, il server chiama SofaScore con gli header corretti, e rigira la risposta. Non è una VPN, non è un servizio esterno — sono 15 righe di codice che girano sul PC dell'utente.

## 3. SERVER PROXY

### server/index.js

```javascript
const express = require('express');
const cors = require('cors');

const app = express();
app.use(cors());

const SOFASCORE_HEADERS = {
  'User-Agent': 'Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/120.0.0.0 Safari/537.36',
  'Accept': 'application/json',
  'Accept-Language': 'it-IT,it;q=0.9,en;q=0.8',
  'Referer': 'https://www.sofascore.com/',
  'Origin': 'https://www.sofascore.com',
  'Cache-Control': 'no-cache'
};

// Proxy tutte le richieste /api/sofascore/* verso www.sofascore.com/api/v1/*
app.get('/api/sofascore/*', async (req, res) => {
  const path = req.params[0];
  const queryString = new URLSearchParams(req.query).toString();
  const url = `https://www.sofascore.com/api/v1/${path}${queryString ? '?' + queryString : ''}`;
  
  try {
    const response = await fetch(url, { headers: SOFASCORE_HEADERS });
    const data = await response.json();
    res.json(data);
  } catch (error) {
    console.error('Proxy error:', url, error.message);
    res.status(502).json({ error: 'Errore nel recupero dati da SofaScore' });
  }
});

// Proxy immagini
app.get('/api/img/*', async (req, res) => {
  const path = req.params[0];
  const url = `https://img.sofascore.com/api/v1/${path}`;
  try {
    const response = await fetch(url, { headers: SOFASCORE_HEADERS });
    const buffer = await response.arrayBuffer();
    res.set('Content-Type', response.headers.get('content-type'));
    res.send(Buffer.from(buffer));
  } catch (error) {
    res.status(502).send('Image proxy error');
  }
});

app.listen(3001, () => console.log('Proxy server running on http://localhost:3001'));
```

### 4.7 Lista partite giocatore

```
GET /api/sofascore/player/{playerId}/events/last/{page}
```

- 30 partite per pagina, `page` = 0, 1, 2...
- `hasNextPage: true/false`
- Filtrare per `event.tournament.uniqueTournament.id` (competizioni selezionate)
- Filtrare per `event.status.code === 100` (partita terminata)

Ogni evento contiene: `id`, `slug`, `startTimestamp`, `tournament` (con `uniqueTournament`), `season`, `roundInfo`, `homeTeam`, `awayTeam`, `homeScore`, `awayScore`, `status`.

### 4.8 Cronaca partita (per i falli)

```
GET /api/sofascore/event/{eventId}/comments
```

Risposta: `comments[]` ordinati cronologicamente inverso. Tipi rilevanti:
- `freeKickLost` → il giocatore ha COMMESSO un fallo
- `freeKickWon` → il giocatore ha SUBITO un fallo e conquistato punizione
- `substitution` → contiene `playerIn` e `playerOut` con ID

Ogni commento ha: `text`, `type`, `isHome`, `player` (oggetto completo con name, id, position, jerseyNumber).

### 4.9 Posizioni medie in campo

```
GET /api/sofascore/event/{eventId}/average-positions
```

Risposta:
```json

  ],
  "away": [...]
}
```

`averageX`: 0=propria porta, 100=porta avversaria. `averageY`: 0=destra, 100=sinistra.

### 4.10 Tornei per paese/categoria

```
GET /api/sofascore/sport/football/categories/all
```

Restituisce tutte le categorie (paesi) con `id`, `name`, `slug`. Usato per trovare gli ID delle categorie per ciascun paese.

### 4.11 Squadre di un torneo/stagione

```
GET /api/sofascore/unique-tournament/{tournamentId}/season/{seasonId}/standings/total
```

Risposta: `standings[0].rows[]` → ogni riga ha `team` con `name`, `id`, `slug`.

### 4.12 Stagioni di un torneo

```
GET /api/sofascore/unique-tournament/{tournamentId}/seasons
```

Risposta: `seasons[]` → prendere la prima (la più recente) per avere il `seasonId` da usare nell'endpoint 4.11.

## 5. ALGORITMO ACCOPPIAMENTO FALLI (CRITICO)

Questo è il cuore del progetto. Verificato su dati reali (Inter-Atalanta 14/03/2026: 16 falli, 14 accoppiati, 2 handball, 0 errori).

### Come funziona

Nei commenti, ogni fallo genera una coppia `freeKickLost` + `freeKickWon` ADIACENTE nella lista. Ma l'ordine tra i due può variare:

```
Caso A (50% dei casi):
  idx N:   freeKickWon  → "Frattesi wins a free kick..."     (VITTIMA)
  idx N+1: freeKickLost → "Foul by Sulemana..."               (AUTORE)

Caso B (50% dei casi):
  idx N:   freeKickLost → "Foul by Zappacosta..."             (AUTORE)
  idx N+1: freeKickWon  → "Mkhitaryan wins a free kick..."   (VITTIMA)
```

### Pseudocodice

```typescript
function extractFoulsForPlayer(comments: Comment[], playerId: number): FoulMatchup[] {
  const results: FoulMatchup[] = [];
  const indexed = comments.map((c, idx) => ({ ...c, idx }));

  for (const comment of indexed) {
    
    // CASO 1: Il giocatore selezionato ha COMMESSO un fallo
    if (comment.type === 'freeKickLost' && comment.player?.id === playerId) {
      
      // Fallo di mano → nessuna vittima
      if (comment.text.toLowerCase().includes('handball')) {
        results.push({ type: 'handball', rawText: comment.text });
        continue;
      }

      // Cercare la vittima: controllare idx-1 E idx+1
      const prev = comments[comment.idx - 1];
      const next = comments[comment.idx + 1];
      let victim = null;
      let zoneText = '';
      
      if (prev?.type === 'freeKickWon') {
        victim = prev.player;
        zoneText = prev.text;
      } else if (next?.type === 'freeKickWon') {
        victim = next.player;
        zoneText = next.text;
      }

      results.push({
        type: 'committed',
        playerFouled: victim,
        zoneText: translateZone(zoneText),
        rawText: comment.text
      });
    }

    // CASO 2: Il giocatore selezionato ha SUBITO un fallo
    if (comment.type === 'freeKickWon' && comment.player?.id === playerId) {
      const prev = comments[comment.idx - 1];
      const next = comments[comment.idx + 1];
      let fouler = null;
      
      if (prev?.type === 'freeKickLost') {
        fouler = prev.player;
      } else if (next?.type === 'freeKickLost') {
        fouler = next.player;
      }

      results.push({
        type: 'suffered',
        playerFouling: fouler,
        zoneText: translateZone(comment.text),
        rawText: comment.text
      });
    }
  }
  return results;
}
```

### Traduzione zone del campo

I commenti API sono SEMPRE in inglese. Tradurre così:
```
"in the defensive half" / "in their own half" → "nella propria metà campo"
"in the attacking half"                       → "nella metà campo avversaria"
"on the left wing"                            → "sulla fascia sinistra"
"on the right wing"                           → "sulla fascia destra"
```

## 6. CONVERSIONE COORDINATE CAMPO

Il campo va disegnato con proporzioni reali: un campo da calcio è 105x68 metri. Usare `aspect-ratio: 68/105` in CSS (più alto che largo).

La squadra di casa va SEMPRE nella metà SUPERIORE, la squadra ospite nella metà INFERIORE.

### Conversione da API a schermo

```typescript
// Per giocatori della squadra di CASA (metà superiore)
function homeToScreen(avgX: number, avgY: number, fieldWidth: number, fieldHeight: number) {
  return {
    x: (avgY / 100) * fieldWidth,
    y: (avgX / 100) * (fieldHeight / 2)
  };
}

// Per giocatori della squadra OSPITE (metà inferiore, specchiati)
function awayToScreen(avgX: number, avgY: number, fieldWidth: number, fieldHeight: number) {
  return {
    x: (1 - avgY / 100) * fieldWidth,
    y: (1 - avgX / 100) * (fieldHeight / 2) + (fieldHeight / 2)
  };
}
```

Nella mappa mostrare SOLO:
- Il giocatore selezionato (evidenziato con colore diverso, es. verde acceso + alone)
- I giocatori coinvolti nei falli di quella partita (pallino + cognome)

NON mostrare tutti i giocatori delle due squadre.

## 7. CALCOLO MEDIE AGGREGATE

```typescript
function calculateStats(statsByTournament: PlayerSeasonStats[]): AggregatedStats {
  const totals = statsByTournament.reduce((acc, s) => ({
    fouls: acc.fouls + s.fouls,
    wasFouled: acc.wasFouled + s.wasFouled,
    minutesPlayed: acc.minutesPlayed + s.minutesPlayed,
    appearances: acc.appearances + s.appearances,
  }), { fouls: 0, wasFouled: 0, minutesPlayed: 0, appearances: 0 });

  return {
    totalFoulsCommitted: totals.fouls,
    totalFoulsSuffered: totals.wasFouled,
    totalMinutesPlayed: totals.minutesPlayed,
    totalAppearances: totals.appearances,
    avgFoulsCommittedPerMatch: (totals.fouls / totals.appearances).toFixed(2),
    avgFoulsCommittedPer90: (totals.fouls / totals.minutesPlayed * 90).toFixed(2),
    avgFoulsSufferedPerMatch: (totals.wasFouled / totals.appearances).toFixed(2),
    avgFoulsSufferedPer90: (totals.wasFouled / totals.minutesPlayed * 90).toFixed(2),
  };
}
```

Se sono selezionate più competizioni (es. Serie A + Conference League), sommare TUTTI i dati prima di calcolare le medie. Se ne è selezionata solo una, usare solo quella.

### 8.5 Responsive card partite

Le card partite nella pagina giocatore usano CSS `columns`:
- Desktop largo (>1200px): `column-count: 3`
- Desktop/tablet (768-1200px): `column-count: 2`
- Mobile (<768px): `column-count: 1`

Ogni card ha `break-inside: avoid` per non spezzarsi tra colonne.

Su desktop le card sono APERTE di default. Su mobile sono CHIUSE di default e si espandono al tocco.

Quando una card viene chiusa, le card sotto nella STESSA colonna salgono a riempire lo spazio. Le altre colonne non si toccano. Quando viene riaperta, le card sotto riscendono.

4. **Panchina e rosa completa** — sotto il campo, tutti i giocatori della rosa che NON sono nella formazione titolare, come tag/chip cliccabili in una griglia wrap. Cliccando → pagina giocatore.

Per la formazione:
- `GET /team/{teamId}/events/next/0` → prendi l'eventId della prossima partita
- `GET /event/{eventId}/lineups` → prendi i giocatori con `substitute: false` (titolari) e la `formation`
- Per il posizionamento sul campo: mappare la formazione (es. "4-2-3-1") a coordinate predefinite. NON c'è un endpoint SofaScore che dà le coordinate dei titolari pre-partita, quindi va fatto con una tabella di mapping formazione → posizioni.

Per la rosa completa:
- `GET /team/{teamId}/players` → tutti i giocatori della rosa

Mostrare nella panchina: (tutti i giocatori della rosa) MENO (quelli nella formazione titolare).

### 9.5 Clic su un giocatore → pagina analisi falli

Vedere sezione 10.

### 9.6 Barra di ricerca

Presente in TUTTE le schermate, sempre in alto nel contenuto. Input con debounce a 500ms.
- Chiama `GET /search/all?q={query}&page=0`
- Mostra dropdown con risultati filtrati per `type === "player"`
- Ogni risultato mostra: foto giocatore (da img API) + nome + squadra + ruolo
- Al clic → naviga direttamente alla pagina analisi falli del giocatore
- Se il giocatore è cercato dalla barra (non dalla navigazione manuale), il sidebar mostra solo il breadcrumb del giocatore

### 10.2 Filtri

**Stagione** — dropdown con tutte le stagioni disponibili. Default: la stagione più recente. Cambiando stagione si ricaricano tutti i dati.

**Competizioni** — checkbox/tag per ogni competizione a cui il giocatore ha partecipato nella stagione selezionata. Default: tutte attive. Cambiando si ricalcolano le medie e si filtra la lista partite.

**Falli commessi / subiti** — due tag toggle. Default: entrambi attivi. Si possono attivare/disattivare indipendentemente. Se disattivi "Falli commessi", le medie dei falli commessi spariscono e nelle card partita non si mostrano i falli commessi (e viceversa).

### 10.3 Card singola partita

Ogni card contiene:

**Header** (sempre visibile, cliccabile per espandere/comprimere):
```
Serie A G.30 · 14 mar
Inter 1 - 1 Fiorentina
```

**Corpo** (espandibile):
```
72 min giocati · Titolare · Uscito al 72'

── Falli subiti (3) ──
45' │ Parisi conquista punizione da Barella nella metà campo avversaria
67' │ Parisi conquista punizione da Mkhitaryan nella propria metà campo
...

── Falli commessi (2) ──
23' │ Parisi commette fallo su Dumfries nella propria metà campo
58' │ Parisi commette fallo su Barella nella metà campo avversaria
...

[Mini campo con posizioni]
```

**Titolarità** — logica:
- `substitute === false` e nessun subOut → "Titolare · 90 min"
- `substitute === false` e subOut → "Titolare · Uscito al {min}'"
- `substitute === true` e subIn → "Entrato al {min}'"
- `substitute === true`, subIn e subOut → "Entrato al {min}' · Uscito al {min}'"

I dati di sostituzione si estraggono dai commenti con `type === "substitution"`.

### 10.4 Giocatore che ha cambiato squadra

Se un giocatore ha giocato per due squadre nella stessa stagione (es. trasferito a gennaio), le partite nella lista vanno separate con un divisore visuale:

```
─── Fiorentina (da gennaio) ─────────────────
[Card partite con la Fiorentina]

─── Empoli (fino a gennaio) ─────────────────
[Card partite con l'Empoli]
```

Per rilevarlo: confrontare il `homeTeam.id` e `awayTeam.id` di ogni partita con l'ID della squadra attuale del giocatore. Se non coincide con nessuno dei due, il giocatore giocava per un'altra squadra.

### 10.5 Nomi giocatori nei falli cliccabili

Ogni nome di giocatore che appare nella lista falli (es. "Barella", "Dumfries") è cliccabile:
- Su desktop: apre lo split view con la pagina di quel giocatore accanto
- Su mobile: naviga alla pagina di quel giocatore

### 10.6 Caricamento dati

1. Al caricamento della pagina: subito le statistiche aggregate (endpoint 4.6) e la lista partite (endpoint 4.7)
2. I dettagli di ogni partita (cronaca falli, posizioni campo) si caricano SOLO quando la card viene espansa (clic su header)
3. Una volta caricati, i dati restano in cache — chiudere e riaprire la card non richiede un nuovo caricamento

## 13. GESTIONE ERRORI E CASI LIMITE

### 13.1 API che restituisce 403
SofaScore può bloccare in qualsiasi momento. Implementare:
- Retry con backoff: 1° tentativo → attesa 1s → 2° tentativo → attesa 2s → 3° tentativo
- Se fallisce dopo 3 tentativi, mostrare messaggio "Dati temporaneamente non disponibili" nella UI (NON crashare)

### 13.2 Partita senza dati cronaca
Alcune partite (amichevoli, leghe minori) non hanno i commenti. L'endpoint `/comments` può restituire array vuoto. Mostrare "Cronaca non disponibile per questa partita" nella card.

### 13.3 Partita senza posizioni medie
L'endpoint `/average-positions` può non avere dati. In quel caso non mostrare la mappa del campo nella card.

### 13.4 Giocatore non trovato nella ricerca
Mostrare "Nessun giocatore trovato" nel dropdown.

### 13.5 Nessun fallo nella partita
Se il giocatore non ha commesso né subito falli in una partita, mostrare comunque la card con il minutaggio ma con testo "Nessun fallo in questa partita".

### 13.6 Minuto del fallo
Il minuto non è un campo separato nei commenti. Va estratto dal testo con una regex tipo:
```typescript
// Dai commenti non è possibile estrarre il minuto direttamente dal testo
// ALTERNATIVA: usare l'ordine relativo dei commenti per ricostruire la cronologia
// I commenti sono ordinati dal più recente al più vecchio
// Se necessario, fare anche GET /event/{id}/incidents per i cartellini che hanno il campo "time"
```

In realtà il modo più semplice è calcolare l'ordine relativo basandosi sulla posizione nell'array dei commenti e sulla sequenza dei commenti nella cronaca, oppure usare il campo `time` presente in alcuni tipi di commento. Se non è possibile estrarre il minuto con precisione, ometterlo e mostrare solo l'ordine cronologico.

## 15. NOTE FINALI PER CLAUDE CODE

1. **Non inventare dati** — tutto deve venire dalle API SofaScore tramite il proxy
2. **Non usare localStorage per stato navigazione** — usa React state/context
3. **Il campo da calcio deve avere SEMPRE aspect-ratio: 68/105** — mai schiacciato o allargato
4. **Tutti i numeri mostrati devono essere arrotondati** — mai decimali lunghi tipo 1.3333337
5. **I colori dei falli sono diversi**: verde per falli subiti, rosso per falli commessi
6. **Lo split view funziona SOLO su schermi sopra 1024px di larghezza**
7. **Le immagini (loghi, foto) vanno tutte tramite il proxy** `/api/img/...` per evitare problemi CORS
8. **I commenti API sono SEMPRE in inglese** — le zone del campo vanno tradotte in italiano nel frontend
9. **L'ordine delle partite è dalla più recente alla meno recente**
10. **Se un giocatore ha cambiato squadra**, separare le partite con divisore visuale indicando il nome della squadra
